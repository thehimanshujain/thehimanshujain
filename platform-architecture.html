High-Risk Architectural Areas
9 risks · Platform-wide
Incentive Checkout Coupling
Critical
incentive-service is called synchronously in the checkout hot path with no circuit breaker from subscription-programs-app or message-service. Single outage blocks orders, membership activation, and email personalisation simultaneously.
checkoutno circuit breaker3 consumers
ML→Pricing Silent Failure
High
ML Toolkit DAG failures cause silent min-price miscalculation in dynamic_pricing — no immediate Grafana alert is configured for DAG failures that affect the Pricing Service API feed. Data quality degradation is invisible to oncall.
machine-learning-toolkitsilent failuremin-price
Bidirectional PCC↔Pricing
Critical
dynamic_pricing calls PCC JTier for ILS scheduling AND PCC JTier calls dynamic_pricing for price resolution — a bidirectional circular dependency that creates cascading failure modes. No anti-corruption layer exists between them.
circular depPCC JTierdynamic_pricing
Message Service Retry Storms
High
message-service has no DLQ for Rocketman delivery failures. Failed messages are retried 3× then silently dropped. Under Rocketman degradation, retry load amplifies to 3× normal outbound traffic — confirmed blast radius for notification systems.
no DLQRocketmanretry amplification
Audience Pool Starvation
High
incentive-service BulkQualifyCassandraCreator runs synchronous per-user SELECT in a 2-thread pool. Under large audience loads, pool starvation blocks the Akka dispatcher and causes AMS WS 30s timeout cascade — confirmed root cause of IS-8782.
2-thread poolCassandraIS-8782
No Redis Lock on Schedulers
High
Both message-service and incentive-service run background jobs on every pod simultaneously with no distributed lock. Under N-pod deployments, N×concurrent audience imports and campaign loggers hit the same Cassandra/GCS write paths — confirmed anti-pattern.
no Redis lockImportAudienceActormulti-pod
Membership God-Object
High
Membership (degree 197, betweenness 0.137) in subscription-programs-app bridges 30+ communities. Any NPE, schema change, or method rename propagates breakage across the entire service graph with zero isolation. Confirmed structural god-object.
Membership.java30+ communitiessubscription-programs
MySQL BoneCP Exhaustion
High
dynamic_pricing uses BoneCP (unmaintained) with minimal partition size. Concurrent bulk requests from TPIS + VIS exhaust the pool before new connections are provisioned — P99 spikes to 80ms. SEVERE Grafana alert firing in us-central1.
BoneCPdynamic_pricing80ms p99
