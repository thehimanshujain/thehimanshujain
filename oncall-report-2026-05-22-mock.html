How It Works

Automated pipeline: OpsGenie signals flow through n8n into squad-level GChat rooms every morning at 08:00 AM IST.

OpsGenie API
Pulls daily alert payload — created, resolved, open, SLA status, MTTR
›
n8n Workflow
Scheduled trigger at 08:00 AM IST, HTTP fetch, JSON transform
›
Alert Scoring
Classifies Red / Yellow / Green per squad thresholds
›
Squad Routing
Maps each service team to its dedicated GChat room ID
›
GChat Delivery
Posts formatted digest; escalates SLA breaches to EM directly

Alert Thresholds

Thresholds used by the n8n scoring node to classify daily squad health status.

Metric 🟢 Green 🟡 Yellow 🔴 Red
Alerts Created (daily) ≤ 5 6 – 10 > 10
Alerts Resolved (same day) ≥ 90% 70 – 89% < 70%
Open Alerts (unresolved) ≤ 2 3 – 5 > 5
SLA Breaches 0 1 – 2 ≥ 3
MTTR (mean time to resolve) < 2h 2 – 4h > 4h
Automation Details

Infrastructure powering the daily reliability digest.

⚡ Powered by n8n + OpsGenie + GChat
🔄 Daily cron at 08:00 AM IST
🗺️ Auto-routes to squad GChat rooms
🚨 Escalates SLA breaches to EM
📈 7-day rolling trend window
🟢 Green / Yellow / Red scoring
📋 JSON audit log retained 30 days
Workflow architecture: n8n self-hosted on internal infra · OpsGenie V2 REST API with team-scoped API keys · GChat Webhooks per squad room · Escalation path: SLA breach → EM DM + squad thread mention · All payloads logged to S3 for 30-day audit retention.
